# Changelog (summary of recent changes)

Date: 2026-05-13

This document summarizes the changes made during the recent work to improve notifications, settings backup/restore, and test stability.

1) Notifications and UX
- Introduced a `Kind` mapping for reminder notifications so each reminder type shows a meaningful small icon.
- Added custom action icons and clearer action labels for reminder notifications (Snooze 1 day, Mark as done, Open reminder).
- Improved notification body copy; moved strings into `strings.xml` for localization.

2) Backup / Restore and security
- Restored the settings UI to rehydrate selections after importing a backup.
- Ensured Gemini API keys are never included in backups or exported JSON:
  - `app/src/main/res/xml/data_extraction_rules.xml` excludes `app_gemini_secret.xml` for cloud and device transfer (Android 12+).
  - `app/src/main/res/xml/backup_rules.xml` excludes the same sharedpref file for older devices.
- `GeminiApiKeyStore` remains local-only and is not serialized into backup payloads.

3) Settings UI
- `AiSettingsCard.kt` changed so the Gemini toggle and Redact Sensitive toggle read directly from `AppSettings` (no stale local state), and connection status shows whether a saved API key exists.

4) Backup/export JSON & tests
- Added/updated `AppDataTransferServiceTest` to verify:
  - settings round-trip in backup JSON
  - exported JSON does not contain the Gemini secret
  - backup JSON parsing behavior for reminders

5) CSV export fixes
- `exportMaintenanceCsv()` and `exportFuelCsv()` (both in-memory and Room implementations) now:
  - sort rows by `dateIso` to provide deterministic output
  - format numeric fields to 2 decimal places (Locale.US) for consistent CSV formatting

6) Test stability fixes
- Updated fragile UI tests to avoid hardcoded dates; tests now use `LocalDate.now().plusDays(...)` where appropriate.

7) Test runs
- Ran unit test suite: all unit tests pass.
- Ran connected instrumentation tests on a local emulator. Two Compose UI tests failed due to an Espresso/Android platform mismatch:
  - Failure: NoSuchMethodException: `android.hardware.input.InputManager.getInstance` during Espresso event injection
  - Likely cause: Espresso/Core test library and the emulator image/platform API are incompatible (Espresso expects a platform method for low-level input injection).

Recommendations to resolve instrumentation failures
- Run instrumentation tests on an emulator/device with an Android system image that matches the AndroidX Test/Espresso supported APIs (try Android 12/13 images).
- Alternatively, update test libraries (Espresso / AndroidX Test) to the latest compatible versions for your SDK/AGP.
- If immediate passing is required, consider marking the affected UI tests as flaky/ignore until the environment or test libraries are aligned.
